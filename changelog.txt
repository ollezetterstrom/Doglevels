Dog Levels — Changelog
======================

1.4.1
=====
Bug fixes:
 - Fixed the Dog Stats GUI XP bar overlapping the stats viewport when a dog
   was below max level. The viewport now reserves space below the title for
   the XP bar + label, so they never overlap.

QoL:
 - Wolves (tamed and untamed) are now attracted to Treats the same way they
   are attracted to bones: holding a Treat makes nearby wolves turn to look
   at you and walk toward you. Implemented by adding a TemptGoal at priority 5
   (matching vanilla's bone TemptGoal) on EntityJoinLevelEvent, with an
   in-memory dedup set so the goal isn't added twice in one session.


1.4.0
=====
Bug fixes:
 - Fixed MAX_LEVEL being cached at class-init time, so editing max_level in
   doglevels-common.toml had no effect until a full JVM restart. The value is
   now read live from config on every call.
 - Fixed DogScreen.predictedBehavior being a static field shared across all
   dogs, so opening dog B's screen after cycling dog A's behavior button
   showed the wrong initial value. Removed; the GUI now reads the synced
   behavior from the server.
 - Fixed a memory leak in DogTickEvents.howlCooldowns: entries were added for
   every wolf that ever howled but never removed. Now cleaned up on
   EntityLeaveLevelEvent.
 - Fixed Splash Bite and AGGRESSIVE target acquisition only hitting mobs that
   extend Monster (missed Slimes, MagmaCubes, Phantoms, etc.). Now uses the
   Enemy interface.
 - Fixed DogScreen using the default locale for String.format, which produced
   comma decimals on EU clients. Now uses Locale.ROOT.
 - All user-facing chat messages moved to the lang file (en_us.json) so they
   can be translated.

Networking:
 - Implemented real server→client sync via a SimpleChannel. The 1.3.2 no-op
   stub has been replaced with DogLevelSyncMessage, which sends the level,
   XP, XP-to-next, max level, and behavior ordinal whenever a dog's state
   changes (level-up, XP gain, treat feed, behavior change) and on
   PlayerEvent.StartTracking so freshly logged-in players see the correct
   state immediately.

GUI:
 - Added an XP progress bar to the Dog Stats screen.
 - The Dog Stats screen now reads live level/XP/behavior from the synced
   cache, with a fallback to the old attribute-derived level for vanilla
   server compatibility.

Config:
 - All ability unlock levels are now configurable (was hardcoded 5/10/15/20/25/30).
 - All ability tuning values are now configurable: Splash Bite radius and
   damage multiplier, Pack Hunter period/radius/buff duration, Howl of
   Vitality chance and heal amount, Lifesteal percent, Fire Resistance and
   Bonded Endurance refresh durations.
 - AGGRESSIVE behavior range is now configurable (was hardcoded 16 blocks).
 - Added safe getters (getInt / getDouble / getBool) that never throw, even
   if the config isn't loaded yet.

Commands:
 - Added /doglevels setlevel <targets> <level>  (admin, OP 2)
 - Added /doglevels addxp <targets> <amount>    (admin, OP 2)
 - Added /doglevels reset <targets>             (admin, OP 2)
 - /doglevels info now works without an entityId — raytraces the wolf the
   player is looking at.
 - Admin subcommands accept entity selectors (@e[type=wolf]) so operators
   can manage multiple dogs at once.

Tests:
 - Ported all three test classes from custom main() runners to JUnit 5, so
   they actually run via ./gradlew test.
 - Added new tests for the 1.4.0 fixes: max-level liveness, behavior cache
   isolation, ability-level config getters, NBT round-trip with new fields.

Internal:
 - Bumped minimum Forge to 52.1.2 (unchanged from 1.3.2).
 - Bumped Java target to 21 (unchanged from 1.3.2).

Deferred to 1.5.0 (intentionally not in this release):
 - Generalization to all tamable animals (cats, parrots, etc.) — large
   architectural change, deserves its own release.
 - New abilities / content additions — content drops should be separate
   from a quality release.


1.3.2
=====
 - Initial public release.
 - Minecraft 1.21.1, Forge 52.1.2.
 - Leveling system with 6 milestone abilities, behavior modes, Treat item,
   /doglevels command, custom GUI.
